# Архитектура папок

- `src/core/` — приложение как оболочка: корневой `App.tsx`, провайдеры (DI, React Query, Ant Design), композиция зависимостей и точка входа UI.
- `src/domain/` — бизнес-модель и контракты (сущности, репозитории, типы). Чистый слой без зависимостей на инфраструктуру или UI.
- `src/application/` — прикладные сервисы/юзкейсы, координируют домен и адаптируются под клиентов. Здесь же React Query хуки, которые вызывают сервисы.
- `src/infrastructure/` — реализации интерфейсов домена (HTTP-клиенты, репозитории, адаптеры). Можно подменять моки/production.
- `src/ui/` — презентационный слой: страницы, компоненты, виджеты. Работает только через application/domain контракты, без прямых обращений к инфраструктуре.
- `src/shared/` — общие настройки/утилиты/типы (например, конфигурация QueryClient).
- `public/` — статические ассеты, доступные как есть во время сборки.
- `index.html` / `src/main.tsx` — точка загрузки Vite: монтирует React, подтягивает провайдеры из `core`.

Рабочий поток по фиче: добавить модель и контракт в `domain` → реализацию/адаптер в `infrastructure` → сценарий/сервис и хук в `application` → UI-страницу/компонент в `ui`, подключив провайдеры из `core`.
